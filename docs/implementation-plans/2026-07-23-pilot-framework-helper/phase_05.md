# Pilot Framework Helper (`@arcjet/ai`) Implementation Plan ÔÇö Phase 5: Example app

**Goal:** A runnable reference app demonstrating the canonical path: route creates context Ôćĺ starts a Vercel Workflow Ôćĺ a step runs `generateText` with a `protectTool`-wrapped, rate-limited tool Ôćĺ a later step performs a mock external action behind `protectAction` Ôćĺ `captureAction` annotates a side effect. Builds in CI; runs locally against a dev Arcjet site.

**Architecture:** New example `examples/nextjs-ai-agent`: Next.js (app router) + `ai@7` + `workflow` (Vercel Workflow DevKit). The model is addressed through the AI SDK's built-in gateway provider (plain model-id string, `AI_GATEWAY_API_KEY` env) so no per-provider package is needed. The `ArcjetAiContext` is created in the route, rides the serialized workflow input, and reaches tools via `toolsContext` ÔÇö exercising the exact serialization boundary the design's explicit-context decision exists for.

**Tech Stack:** Next.js 16.2.6, React 19, `ai` ^7.0.36, `workflow` ^4.6.1 (verified GA on npm, `vercel/workflow`), `zod` ^4.4.3 (idiomatic tool `inputSchema` for end users), `@arcjet/ai` + `@arcjet/guard` via `file:../../` (repo convention).

**Scope:** Phase 5 of 6 from `docs/design-plans/2026-07-23-pilot-framework-helper.md`.

**Codebase verified:** 2026-07-23. Key facts: examples reference workspace packages as `"file:../../<dir>"`; newest Next.js example (`examples/nextjs-react-hook-form`) uses Next 16.2.6, scripts `dev/build/start/typecheck` (`tsc --noEmit`), `.env.local.example`; example CI = `.github/workflows/reusable-examples.yml` `node-examples` job matrix (add the folder name; CI runs root `npm ci && npm run build`, then example `npm ci`, `npm run build --if-present`, `npm run typecheck` ÔÇö no tests); dependabot needs NO changes (version updates disabled; Renovate discovers examples automatically); no existing example uses `ai` or `workflow`. Workflow DevKit facts (official docs, verified against npm): `withWorkflow` from `workflow/next` wraps next.config; directives are function-level `"use workflow"` / `"use step"`; `start(workflowFn, [args])` from `workflow/api` returns a run handle (`run.id` ÔÇö the full `Run` shape was not confirmed from primary sources; treat other properties as unknown); arguments/returns must be JSON-serializable; local dev needs only `next dev`; inspect runs with `npx workflow inspect runs` / `npx workflow web`.

**Known gap (inherited from Phase 3's capture finding):** until
`experimental_capture()` merges into `@arcjet/guard` (it currently lives on
`origin/quinn/experimental-capture` only), a local run produces correlated
guard **decisions** but no capture **events** ÔÇö `@arcjet/ai` warns and
skips them. The README must state this plainly. AC5.2's "decisions and
events" is fully satisfiable only after guard ships capture; verify
decisions now, re-verify events when capture lands.

## Scenario

A tiny "support agent": the user asks a question about an order; the agent
may call a `lookupOrder` tool (rate-limited ÔÇö the model can burn through a
token bucket), then the workflow updates a (mock) ticket behind
`protectAction`, and `captureAction` records that a (mock) notification went
out. Everything risky is guarded; everything interesting is captured.

### Task 2: Workflow, tools, and route

**Files:**
- Create: `examples/nextjs-ai-agent/workflows/support-agent.ts`

**Step 1: `workflows/support-agent.ts`**

```ts
import {
  aiToolsContext,
  captureAction,
  protectAction,
  protectTool,
  securityMetadata,
} from "@arcjet/ai";
import type { ArcjetAiContext } from "@arcjet/ai";
import { slidingWindow, tokenBucket } from "@arcjet/guard";
import { generateText, stepCountIs, tool } from "ai";
import { z } from "zod";
import { arcjet } from "@/lib/arcjet";

export interface SupportAgentInput {
  question: string;
  arcjet: ArcjetAiContext;
}

// Rule configs are created once at module scope; inputs per call.
const lookupLimit = tokenBucket({
  bucket: "order-lookups",
  refillRate: 5,
  intervalSeconds: 60,
  maxTokens: 10,
});

const ticketLimit = slidingWindow({
  bucket: "ticket-updates",
  maxRequests: 5,
  intervalSeconds: 60,
});

const baseMetadata = securityMetadata({
  agent: "support-agent",
  workflow: "support-request",
});

const tools = {
  lookupOrder: protectTool(
    arcjet,
    tool({
      description: "Look up an order by its number.",
      inputSchema: z.object({ orderNumber: z.string() }),
      async execute({ orderNumber }) {
        return lookupOrderRecord(orderNumber);
      },
    }),
    {
      action: "order.looked-up",
      rules: () => [lookupLimit({ key: "demo-user", requested: 1 })],
      metadata: ({ orderNumber }) =>
        securityMetadata({ resource: `order:${orderNumber}` }),
    },
  ),
};

export async function supportAgentWorkflow(input: SupportAgentInput) {
  "use workflow";
  const answer = await stepRunAgent(input);
  await stepUpdateTicket(input, answer);
  return { answer };
}

async function stepRunAgent(input: SupportAgentInput) {
  "use step";
  const result = await generateText({
    model: "anthropic/claude-haiku-4-5",
    instructions:
      "You are a support agent. Use the lookupOrder tool for order questions. " +
      "If a tool call is denied by security policy, do not retry it; explain to the user instead.",
    prompt: input.question,
    tools,
    toolsContext: aiToolsContext(input.arcjet, tools),
    stopWhen: stepCountIs(3),
  });
  return result.text;
}

async function stepUpdateTicket(input: SupportAgentInput, answer: string) {
  "use step";
  await protectAction(
    arcjet,
    input.arcjet,
    {
      action: "ticket.updated",
      rules: [ticketLimit({ key: "demo-user" })],
      metadata: {
        ...baseMetadata,
        destination: "internal",
        reversibility: "reversible",
      },
    },
    async () => {
      // Mock external action: a real app would call its ticketing system.
      console.log("ticket updated with answer:", answer.slice(0, 80));
    },
  );

  captureAction(arcjet, input.arcjet, {
    action: "notification.sent",
    metadata: { ...baseMetadata, destination: "internal" },
  });
}

// Mock order store.
function lookupOrderRecord(orderNumber: string) {
  return {
    orderNumber,
    status: "shipped",
    carrier: "ACME Post",
    eta: "2 days",
  };
}
```

Implementor notes:
- The exact rule-builder option names above (`bucket`, `refillRate`,
  `intervalSeconds`, `maxTokens`, `maxRequests`, and the input shape
  `{ key, requested }`) must be checked against
  `arcjet-guard/src/rules.ts` / the `@arcjet/guard` README at implementation
  time and corrected if they differ ÔÇö the guard client in this repo is the
  source of truth. Same for whether `slidingWindow` inputs take `{ key }`.
- If `protectAction`'s denial should stop the ticket update but not the
  workflow, wrap the call in `try/catch (ArcjetDeniedError)` ÔÇö for the
  example, letting it throw is fine and shows the enforce-by-default
  posture; add a one-line comment saying a real app would catch it.
- `generateText`'s `instructions` line doubles as the skill file's
  recommended denial-handling system-prompt line (Phase 6 references this
  example).

**Step 2: `app/api/agent/route.ts`**

```ts
import { createAiContext, securityMetadata } from "@arcjet/ai";
import { start } from "workflow/api";
import { NextResponse } from "next/server";
import { supportAgentWorkflow } from "@/workflows/support-agent";

export async function POST(request: Request) {
  const { question } = await request.json();

  // One context per run; its correlation ID joins every guard decision and
  // capture event this run produces. Pass an existing ID (e.g. a ticket or
  // request ID) instead to join Arcjet data to your own systems.
  const ctx = createAiContext({
    metadata: securityMetadata({
      agent: "support-agent",
      workflow: "support-request",
    }),
  });

  const run = await start(supportAgentWorkflow, [
    { question, arcjet: ctx },
  ]);

  return NextResponse.json({
    runId: run.id,
    correlationId: ctx.correlationId,
  });
}
```
